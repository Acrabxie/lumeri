# Lumeri 模块流式布局

Lumeri v3 工作区使用确定性的 **连续比例 + justified flow**。它不是可见的列网格：模块按自然行流动，每一行自动吃满可用宽度，所有行再共同吃满工作区高度。关闭、移动或缩放模块后，其他模块会立即补位，因此除有意保留的 8px 呼吸缝外不会出现空洞。

## 数据

每个可见模块只有三个布局字段：`id`、`width`、`height`。`width` 和 `height` 都是连续百分比权重，不是整数格数。顺序与尺寸仍分别持久化为：

- `lumeri:v3:workspace-order`
- `lumeri:v3:workspace-sizes`

旧版保存的 `cols` / `rows` 会在读取时换算成百分比，不会丢失用户原来的大小偏好。

## 铺满规则

1. 按持久化顺序读取可见模块，根据连续宽度权重形成自然行；时间线等接近全宽的模块独占一行。
2. 每行先保障模块可操作的最小宽度，再按用户宽度权重分配剩余空间。
3. 每行最后一个模块吸收浮点余量，确保行的左右边缘精确贴合工作区。
4. 所有行以用户高度权重共同分配可用高度；最后一行同样吸收浮点余量，确保底边贴合。
5. 工作区尺寸变化时由 `ResizeObserver` 重新计算，面板自动补位且保持无重叠。

## 交互

- 拖动标题栏：改变模块顺序，松手后平滑流动到新位置。
- 拖动右边界：连续调整宽度权重，相邻模块实时接管释放的空间。
- 拖动下边界：连续调整高度权重，其他行同步补齐。
- 拖动右下角：同时调整宽度和高度。
- 边界获得键盘焦点后，方向键每次调整 2%，按住 Shift 时每次调整 5%。
- 拖拽或缩放过程中只改变布局数据，不重建模块内部状态；结束时写入 localStorage。

算法实现在 `static/v3/workspace-layout.js`，为纯函数，可在浏览器和 Node 测试中复用。测试会随机生成顺序和连续尺寸，逐行验证左右贴边、整体底边贴合以及模块零重叠。
