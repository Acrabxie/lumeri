# Lumeri 模块放置算法

Lumeri v3 工作区使用确定性的 **12 列占用矩阵 + 行优先 first-fit**，统一处理初始排版、模块拖拽和边界缩放。

## 数据

每个可见模块只有三个布局字段：`id`、`cols`、`rows`。顺序与尺寸分别持久化为：

- `lumeri:v3:workspace-order`
- `lumeri:v3:workspace-sizes`

默认尺寸：预览 `8×6`、时间线 `12×4`，大纲/后台任务/文件/会话 `4×3`。尺寸始终经过每类模块的最小/最大跨度约束。

## 放置

1. 按持久化顺序读取可见模块。
2. 对当前模块从第 1 行第 1 列开始，先从左到右、再从上到下扫描。
3. 只有目标矩形内全部单元格为空时才落位，并一次性标记整个矩形。
4. 没有空位则继续增加虚拟行；网格容器允许纵向滚动。
5. 所有模块落位后，用实际最大行号设置 CSS 网格行数。

由于“检查整块为空”和“占用整块”在同一个同步步骤内完成，任意两个模块都不可能占用同一单元格。相同的顺序与尺寸必定产生相同布局。

## 交互

- 拖动模块标题栏：比较指针相对落点模块中心的横纵距离，以更明显的方向决定插入其前或后，然后重新 first-fit 打包。
- 拖动右边界：把水平像素差量量化为列跨度。
- 拖动下边界：把垂直像素差量量化为行跨度。
- 键盘聚焦边界后，可用方向键逐列/逐行调整。
- 拖拽或缩放过程中只改变布局数据，不搬动模块内部状态；结束时写入 localStorage。

算法实现在 `static/v3/workspace-layout.js`，为纯函数，可在浏览器和 Node 测试中复用。
