# 前端协作规范 · PROTOCOL

**Claude Code 与 Codex 都必须遵守。** 这是硬约束;与个人习惯冲突时,以本文件为准。
(规则先立清楚;具体任务的分析与方案后期共同补充。)

## 一、适用范围
- 对象:Lumeri 前端(static/v3/ 及其后端契约面)的一切理解、记录、优化。
- 参与:Claude、Codex(两个 agent 都读都写)、Acrab(审阅)。
- 单一事实源:`docs/frontend/`;任务看板 = `BOARD.md`。

## 二、五条铁律(违反即回退重来)
1. **先读后写**:动手前必读 `BOARD.md` + 本规范最新版。
2. **独占认领**:一次只认领并只改自己那几行;**禁止改动对方标 🟡「处理中」的行**。
3. **实测才算完成**:前端改动完成必须在 http://127.0.0.1:7788 **硬刷新实测通过**;禁止只凭"读代码 / 测试绿"就标 ✅。
4. **协议面双端同步**:凡碰 event kind / error code / ask control / `/sessions*` 形状,必须 web + CLI(lumeri-cli)+ 双端测试**同一个 commit** 落地,否则不算完成。
5. **写完即校验**:每次写文件后核对字节数 / 尾行;进展日志 **append-only,禁止改旧行**。

## 三、任务生命周期
新增 → 认领(⬜→🟡 + 填负责人)→ 处理 → 完成(🟡→✅ + 补方案/验收)或 阻塞(→⛔ + 标依赖)。
每一步都在 `BOARD.md` 底部进展日志追加:`- [日期 署名] 动作`。

## 四、状态语义
| 状态 | 含义 | 谁能设 |
|---|---|---|
| ⬜ 待处理 | 无人认领 | 任何人新增 |
| 🟡 处理中 | 已认领,负责人**独占该行** | 认领者 |
| ✅ 完成 | 已按铁律 3/4 实测验收 | 认领者 |
| ⛔ 阻塞 | 等依赖(依赖列写「等 FE-XXX」) | 认领者 |
| ⏸️ 搁置 | 暂不做,须注明原因 | 认领者 |

## 五、写入纪律(防撞车 + 防丢失)
- 改前**重读最新版**,基于最新内容改,不凭记忆改。
- **小步写、写完立即校验**;大文件分块,禁止攒一大批再写(曾因输出截断丢过文档)。
- 只 `git stage` 显式改动路径,**禁止 `git add -A` / `git clean`**(主树有多 agent 并发改动)。
- 本地 commit 身份 **Acrabxie**;**禁止 push 公开 origin**。
- 注:Claude 端沙箱写不了外置盘(exFAT 无扩展属性),改 `docs/frontend/` 或 gemia 源码需禁用沙箱;Codex 不受此限。

## 六、职责与协同
- 理解知识库(`understanding/`)主由 Claude 维护,Codex 可读可补。
- 功能改动**同步 lumeri-cli**(web 与 CLI 功能对等)。
- **不动**常驻服务:live :7788 sidecar、FlClash、像素螃蟹 :7799 等,只读诊断。

## 七、新增任务格式
- 看板加行:`| FE-0XX | 问题一句话 | P0/P1/P2 | ⬜ | — | — | file:line |`
- 简要方案写进 BOARD「方案详情」;深度分析写进 `understanding/10-assessment.md`,BOARD 只留指针。
