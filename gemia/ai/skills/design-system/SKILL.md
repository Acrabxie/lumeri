# design-system

## 何时使用

用户在意"整片好不好看、专不专业、风格统不统一",而不只是某一步操作。我提供**跨所有创作步骤都该守的质量基准**,并在出片前做**红旗自检**。执行细节交给对应领域 skill(见文末分工)。

## 核心基准:克制的电影感

成片应"冷静、通透、有呼吸",而非高饱和满屏转场的"AI 感"。四条底层原则贯穿:
- **科技/设计要隐形自然**——观众只觉"好看",看不出特效/AI 痕迹。
- **Less is more**——每个转场/运动/调色都要有理由,能不加就不加。
- **设计正确优先于商业恰当**——"更炫"与"更对"冲突时选对的。
- **尊重疯狂提案**——有明确艺术意图可破例,但要写清意图。

## 双层规范:静态 + 时间

Lumeri 的视觉身份用 DESIGN.md 双层 token 描述(见 `references/DESIGN.md` 的 YAML front-matter,是唯一规范值):
- **静态层**:colors / typography(字幕)/ frame(画幅安全框)/ grading(调色)/ depth(景深)
- **时间层**(视频特有):motion(运动/缓动)/ transition(转场纪律)/ pacing(节奏)/ audiosync(音画)

## 交互行为红旗(每轮都守;蒸馏 PAIR/HAX,全文 `references/针对性/06-AI交互准则.md`)

- **存疑降级**:模糊请求(如"调一下节奏")→ elicit 给 2-4 选项+推荐默认,或只做最小可逆改动并明说;**禁静默大改**。
- **提问纪律**:只在轮次边界问;可跳过(跳过=按声明默认继续);本会话已给过的信息不重复索要。
- **解释一行**:每个动作带 ≤1 行"为什么"(用户原话或手册规则);置信度只说 确定/尚可/存疑 三档,禁裸百分比。
- **失败三件套**:出了什么事+下一步+现成动作(含"到时间轴手动改"的退路);禁静默吞错;认错谦逊+说边界。
- **高风险过闸**:不可逆/扣额度/对外发布/删素材源 → 必过 plan 审批,批准前枚举具体 op;改动逐 op 可撤。
- **文案**:讲收益不讲技术(禁 神经网络/LLM/大模型 等黑话);禁"总是/保证/100%"式全能宣称。

## 出片前红旗自检(命中即打回重做)

- **调色**:全局饱和 ≤ 0.85;肤色贴肤色线且额外降饱和;不死黑不爆白;跨镜头色温差 ≤ 300K;拒绝全程 teal-orange。
- **节奏**(最大杠杆):ASL 随情绪走,时长要有长/中/短/超短分布;禁止 ≥4 连续同时长镜头(节奏平坦=机器感)。
- **字幕**:字体家族 ≤2、贴下方安全区不居中不盖脸、出入用缓动、对比度 ≥4.5:1。
- **构图**:三分线而非居中;连续 ≥3 居中/对称要重构;景别有跨度;运镜必须指向具名元素。
- **转场**:默认直切;全片转场数 ≤ 镜头数 × 0.20;每个转场要有理由;禁连续两转场/炫技。
- **音画/一致性**:音画对齐 ≤100ms、关键点踩拍;首尾视觉身份一致;响度 -14LUFS。

完整红旗表与理据见 `references/辅助指导手册.md`。

## 自主策略

- 需要判断画面现状时,先用 `get_metadata` 或 `video_summarize` 读帧/摘要,再对照红旗表自审。
- 我负责"定基准 + 出片前把关",不直接执行操作:把具体调色/剪辑/字幕/构图交给上表对应 skill。
- 每一步写自然的 `assistant_message` 说明当前在守哪条质量基准,不输出内部推理。

## 边界 / fallback

何时不用我:用户只要单点操作(裁一刀/暖一点/加个字幕/换个转场)时,直接用对应执行 skill,不必加载我。我只在"整体设计/质量/一致性/成片自检"语境下出场。
