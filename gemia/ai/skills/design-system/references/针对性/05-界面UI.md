# 针对性指导手册 · 界面UI(Lumeri Product UI)

> 管 **Lumeri 产品界面本身**(web v3 / CLI / 桌面壳),不管成片画面(那是
> [`DESIGN.md`](../DESIGN.md) 的事)。身份三件套不动摇:**无边框**(分隔=表面梯+阴影+留白)、
> **冰蓝只做点睛**、**香肠胶囊母题**。本册把四大权威源蒸馏为可执行规,token 见上方 frontmatter。

## 1. 表面与层级:亮度爬梯,不是画框

- 分隔一律用 `{ui.surface.ladder}` 相邻梯级 + `{ui.elevation.*}` 阴影 + 留白;**全站零 1px 分隔线**(焦点环除外)。
- 浮层永远比被盖表面**更亮**(暗色模式"抬升=变亮",HIG/M3 同源);弹出菜单坐 s3 + `{ui.elevation.L2}`。
- 拒绝 M3 的 surface-tint(primary 染层)——会把 UI 染成冰蓝雾,用纯明度梯。

## 2. 交互反馈:状态层膜,不改底色

- hover/press/drag = 在元素上叠**前景色**半透明膜(`{ui.state.*}`),用 `::after + currentColor` 或
  `color-mix()` 实现;反馈不得引起任何布局位移。
- 按压膜 **0ms 进入**、~200ms 退出;拖拽中叠 16% 膜 + 升 `{ui.elevation.L4}`。
- 禁用态 = 前景 38% + 容器 10% 双数字,**不是变灰**。

## 3. 焦点与键盘:无边框也要全键盘

- 焦点环用 `{ui.focus.ring}`(outline 跟随 border-radius,胶囊不破形),仅 `:focus-visible`。
- 弹层三契约(Radix 蒸馏):**菜单**=箭头 roving + Enter 激活 + Esc 回触发器;**Popover(混合控件面板,
  如"+"菜单)**=面板内 Tab 导航 + Esc 回触发器 + pointerdown 外点关;**Dialog**=焦点困笼 + 背景 inert +
  Esc 逐层退出(维护 open-layer 栈)。
- 自制开关四件套:`role="switch"` + `aria-checked` + `tabindex="0"` + Space/Enter 都能切。
- 弹层开/关状态只存一处,`setOpen()` 同步写 aria-expanded、`data-state`、hidden 三者,禁止漂移。

## 4. 动效:短而准,可打断

- 时长只取 `{ui.motion.duration}` 档位;**进出不对称**(出场 ≤ 进场 60%);淡入淡出配对 150/75ms。
- 弹层进场按 `{ui.motion.popover}`,transform-origin 锚在触发点;贴视口边缘时翻转 side 并同步
  `data-side` 让滑入方向跟着翻。
- 高频交互(hover/流式 token)**不加动效**;所有动画可被打断;`{ui.motion.reduced}` 必须实现。

## 5. 文字:一族字体,一把阶梯

- 家族 ≤ `{ui.type.families-max}`;字重白名单 `{ui.type.weights}`;字号地板 `{ui.type.floor}`。
- 层级=字号+字重+四级灰阶(--text-1..4,每级对其表面 ≥7/4.5/3/3:1),**不靠加框加色**。
- 强调按 `{ui.type.emphasis}`;标题按 `{ui.type.tracking}` 收字距(一行 CSS 买一半"苹果质感")。

## 5.5 图形与图标:图案优先

- **文字能被图案代替,图案优先**(产品原则第 6 条):按钮/开关/状态尽量用图标承义,文字退居
  tooltip 与 aria-label;例外=不可逆动作的确认按钮(必须留文字,防误触)。
- 规格从 `{ui.icon}`:24 网格、1.6 描边、round caps、currentColor;全站取自 `{ui.icon.sprite}`
  单一 sprite,禁止各处内联重复画。
- 图标与文字并排时,描边粗细与字重视觉匹配(HIG);图标独用时热区仍 ≥ `{ui.target.min}`。
- 无障碍按 `{ui.icon.a11y}`:图案代替文字≠代替可访问名称。

## 6. 颜色语义:一色一义

- 冰蓝=可交互/品牌/运行态**专用**;成功恒绿、失败恒红、警告恒琥珀;同色不承载两义。
- 状态指示 = 色 + 字/形**双通道**,禁止色点独行(色盲不可读)。
- 冰蓝填充控件文字一律 `{ui.brand.ink-on-ice}`——**白字压冰蓝 1.98:1 = 硬失败**。

## 7. 反馈分级:成功默认静默

- 被动状态就地显示(工具卡 chip 变色即可);toast 只留给导出级事件;**模态只配不可逆破坏**。
- 可撤销操作(timeline ops 覆盖的一切)**零确认弹窗**;不可逆操作必确认且写明后果。
- 失败态必含:一句原因 + ≥1 个可点恢复动作;不存在裸"失败"终态。
- 任何用户动作 <100ms 给可见响应;>2s 等待给进度;渲染期间聊天/输入保持可用。

## 8. 尺寸与词表纪律

- 指针目标 ≥ `{ui.target.min}`(热区可用 `{ui.target.expand}` 隐形扩);菜单行 ≥ `{ui.target.menu-row}`。
- 拖拽必配点击替代(WCAG 2.5.7):scrub=点轨道定位、clip 移动=选中+按钮、修剪=数字入出点。
- 组件状态只许 `{ui.data-state.*}` 词表驱动;CSS 里禁裸 hex、禁裸 px 间距、禁第四种圆角——
  一切取值走 var(--\*)。**约束集越小,agent 生成的 UI 越不漂。**
- 不抄清单:SF 字体/毛玻璃材质/弹性滚动/12 系统色/ripple 涟漪/spring 物理 token(vanilla 无解,用
  `{ui.motion.enter}` 近似)。

## 9. Agent 自检 prompt(生成/修改 UI 后逐条跑)

```
界面审查(命中任一→修正复检):
1. 新样式有无 1px 分隔边框?浮层是否比被盖表面更亮、带正确档位阴影?
2. 文字对实际背景 ≥4.5:1?冰蓝填充上是否深墨字?次级灰 ≥#8b949e?
3. 仅键盘能否走通全流程?Tab 处处有 2px 冰蓝 offset 环?Esc 逐层退出且焦点回位?
4. 动画是否全部 ≤350ms、出快于进、可打断?reduced-motion 下退化为纯 fade?
5. 可点目标 ≥24px?拖拽有点击替代?开关有 role/aria-checked/Space+Enter?
6. 字号 ≥10px、字重 ∈{400,500,600,700}、字体家族 ≤2?
7. CSS 有无裸 hex/裸间距/第四种圆角/.is-open 式类名?状态是否走 data-state 词表?
8. 成功是否静默、失败是否带原因+恢复动作、不可逆是否有确认?
```
