# 针对性指导手册 · 构图运镜(Composition & Camera)

> 引用 [`DESIGN.md`](../DESIGN.md) 的 `{frame.*}` / `{depth.*}` / `{motion.*}` token,
> **不重复数值**。目标:让画面有透视、有导演意图,而非"模板生成的居中截图"。

## 1. 构图:三分而非居中(P1)

- 关键元素落在**三分线**(约 1/3 或 2/3 处),而非死居中;非对称平衡更有张力。
- **禁单调**:连续 ≥3 个居中/对称构图会显"无导演意图";主体位置要有变化。
- 视线留白:人物视线方向一侧多留空间(look room),头顶留合理 head room。
- 所有主体落在 `{frame.action-safe}` 内,双幅(横 `{frame.aspect-horizontal}` / 竖 `{frame.aspect-vertical}`)分别构图。

## 2. 景别:有跨度地组接

- 一个序列要覆盖**远/中/近**:建立镜头(远)→ 中景 → 特写,信息自然递进。
- 连续 3 镜至少一次**景别或视角变化**(俯/平/仰、左/正/右);镜头单调是视觉疲劳主因。
- 特写强而不滥,单镜别过长(参考主册 `{pacing.cu-max}`),否则显诡异。

## 3. 景深:分层造纵深

- 按用途选光圈:特写/情感 `{depth.dof-shallow}`、对话中景 `{depth.dof-medium}`、
  环境建立 `{depth.dof-deep}`。
- **前中后景分工**:前景压虚做框、焦点主体锐、背景带虚化拉开纵深;全平面 = 廉价摄像头感。
- **焦点转移**(focus pull)以 `{depth.focus-pull-duration}` 为默认节奏,用来把视线从 A 带到 B。

## 4. 运镜:每一次运动都要有指向

- **必须有目标**:相机推拉/摇移遵循 `{motion.camera-move-rule}`——指向画面中一个**具名**的
  关键元素;无目的的 Zoom/Dolly 让观众困惑"镜头为什么动"。
- **Ken Burns 克制**:静帧缓推/缓移单次 ≤ `{motion.ken-burns-max}`;方向与叙述/音乐节拍同向。
- **曲线不用 linear**:运动用 `{motion.easing.move}`,弹性 `{motion.easing.dramatic}` 慎用于特殊强调。
- 运镜服务叙述:引导视线、揭示信息、跟随动作——三者之一,否则不动。

## 5. 连续性(镜头之间的平顺)

- 出点入点要衔接:前一镜的运动/视线方向,与后一镜的入场方向协调,避免"跳"。
- 遵守 180° 轴线与视线匹配(详见《剪辑节奏》§3),构图层面保证人物左右位置稳定。

## 6. Gemini 自检 prompt(植入构图/运镜工作流)

```
构图运镜审查(逐镜 + 序列):
1. 主体是否在三分线而非居中?连续 ≥3 镜是否都居中/对称 → 重构其一。
2. 本序列景别是否覆盖远/中/近?连续 3 镜是否至少一次景别/视角变化?
3. 景深是否按 {depth.dof-*} 选择?前中后景是否分层?
4. 每个运镜是否满足 {motion.camera-move-rule}(有具名指向)?Ken Burns 是否 ≤ {motion.ken-burns-max}?
5. 相邻镜头出入点方向是否衔接?是否守 180° 轴线?
命中任一 → 调整并说明修改点。
```
