## 一、Token Schema 速览:双层结构

Lumeri Video 在 Google DESIGN.md 基础上做了**双层 token** 扩展:

| 层 | token group | 来源 | 作用 |
|----|-------------|------|------|
| 静态层 | `colors` `typography` `frame` `grading` `depth` | 沿用/扩展 DESIGN.md | 描述"静止一帧"长什么样 |
| 时间层 | `motion` `transition` `pacing` `audiosync` | **Lumeri 新增** | 描述"帧与帧之间"如何演变 |
| 组合层 | `components` `quality-gate` | 组合上两层 | 可复用模板 + 交付硬门 |

**引用语法**:`{group.token}` 或 `{group.sub.token}`,例如 `{grading.presets.cinematic}`。
所有 `.md` 文件只能引用,不得复写数值——这是 [DESIGN.md 规范] 的第一底线。

### Token 类型约定
- **Color**:任意合法 CSS 颜色(hex / rgb / oklch);内部统一转 sRGB 做 WCAG 对比度校验。
- **Dimension**:带单位字符串(`px` / `em` / `rem` / `s` / `ms` / `K` / `LUFS`)。
- **Scalar**:0–1 归一化数(饱和、安全框比例、时长分布)。
- **Preset 对象**:一组命名参数(如 grading preset 的 lift/gamma/gain/sat/temp)。
- **Reference**:`{path.to.token}`,组件层允许引用整组。

## 二、红旗自检表(Red Flags · Gemini 出片前必跑)

> 命中任一条即**打回重做**,不得交付。这是把"能出片"顶到"专业"的工程护栏。

### A. 视觉 & 调色
- [ ] 连续 3 个镜头构图都居中/对称 → 重构其一
- [ ] 全局饱和 > `{grading.saturation-global-cap}` → 降饱和
- [ ] 冷暖色温切换无过渡(硬切) → 插入过渡帧
- [ ] 字幕/文字与背景对比度 < 4.5:1(WCAG AA) → 调色或加描边
- [ ] 单镜色阶只占画面 <50% 动态范围 → 调 Curve,拉开层次
- [ ] 出现死黑(<`{grading.black-point-floor}`)或爆白(>`{grading.white-point-ceiling}`) → 收两端

### B. 文字 & 排版
- [ ] 字幕不在下方安全区 / 居中 → 重新定位
- [ ] 同帧 ≥3 字体或字重 → 合并为统一体系
- [ ] 文字出入无 easing → 加 `{motion.text-enter}` / `{motion.text-exit}`
- [ ] 字幕时长 <0.3s 或明显长于该镜内容 → 调整
- [ ] 标题字体与题材气质无关(蛮荒题材用 Comic Sans 之类) → 改字体

### C. 运动 & 转场
- [ ] 转场总数 > 镜头数 × `{transition.budget-ratio}` → 删减
- [ ] 连续 2 个转场无间断 → 至少留 2 帧直切/纯色
- [ ] 单镜内 ≥2 个转场效果 → 去其一
- [ ] Zoom/Dolly 无具名指向对象 → 删除或改为"看向"某元素
- [ ] Ken Burns > `{motion.ken-burns-max}` → 压短

### D. 音画同步 & 节奏
- [ ] 字幕与口型差 > `{audiosync.lipsync-tolerance-ms}` → 重新对齐
- [ ] 音乐节拍与剪辑点无关联(不踩拍) → 找关键帧重剪
- [ ] 触发 `{pacing.flatness-guard}`(≥4 连续同时长镜头) → 插长/超短镜打破
- [ ] 背景音乐情绪与文案/画面语气不符 → 换乐或改语气

### E. 一致性 & 身份
- [ ] 首场与末场字体/色温/滤镜明显不同 → 统一套用身份 token
- [ ] 同类元素(所有标题/所有下三分标)样式不一 → 建 token 并套用
- [ ] 色温在冷/暖之间随意摇摆 >3 次 → 定义冷暖区段,减少切换

### F. 导出 & 交付
- [ ] 色彩空间未锁定 → 明确 Rec.709(SDR)/ DCI-P3(DCP)
- [ ] 分辨率/画幅与目标平台不符(竖屏用 16:9 导出) → 改目标尺寸
- [ ] 响度未归一 → 归一化到 `{audiosync.loudness-target}`

## 三、Do's / Don'ts 全表(可直接进 system prompt)

### Do ✅
| 条款 | 域 | 说明 |
|------|----|------|
| 择一字体家族贯穿全片 | 排版 | 标题+正文各一,最多加一衬线强调 |
| 保持冷暖分区、成段过渡 | 调色 | 段落级色温,而非逐镜乱飘 |
| 景别有跨度 | 构图 | 连续 3 镜至少一次景别/视角变化 |
| 音乐节拍踩剪辑点 | 剪辑 | 关键转场落在鼓点/和弦转换 |
| 关键信息留呼吸空间 | 排版 | 字幕不盖脸、标题不盖风景 |
| 渐进式引导视线 | 构图 | 缓推/摇移把视线带向"故事核心" |
| 文字出入用缓动 | 动画 | ease-out 入、ease-in 出 |
| 音乐情绪匹配语气 | 音画 | 危险=低沉冷调,起航=明快暖调 |

### Don't ❌
| 条款 | 后果 |
|------|------|
| 同帧 3+ 字体 | 层级混乱、显廉价 |
| 转场超 `{transition.budget-ratio}` 预算 | 碎片化、ADHD 剪辑 |
| 连续镜头时长接近(±0.3s) | 节奏平坦、机器感 |
| 字幕居中屏幕 | 无主次、非专业 |
| 过饱和(>`{grading.saturation-global-cap}`) | 刺眼、廉价 |
| 运动无指向 | 观者困惑镜头为何动 |
| 冷暖 <1s 快切 | 视觉割裂、晕眩 |
| 音画对齐差 >100ms | 像后期硬补字幕 |
| 色阶占屏 <50% | "廉价摄像头感" |
| 构图始终居中对称 | 模板感、无导演意图 |

## 四、理据库(为什么这些数值是这些值)

> 正文规范里"不解释数值",理据集中放这里,供 Gemini 需要判断"能否突破条款"时查阅。

- **饱和上限 0.85**:超过此值人眼开始感知"滤镜/贴纸感",色彩边界毛躁;专业片靠留白与降饱和让视线休息。
- **肤色额外 -0.20**:即便整体鲜活,肤色一旦与背景同饱和就"显假";矢量示波器上肤色点须贴肤色线。
- **黑点 3% / 白点 98%**:两端各留安全边界,既避免死黑爆白,也给交付端色域转换留裕量。
- **色温容差 300K**:>300K 的跨镜漂移人眼可辨,破坏"同一场景同一时空"的可信度。
- **转场预算 0.20**:国际级作品转场极少且战略;经验阈值 = 镜头数的 1/5 封顶。
- **ASL 分情绪**:节奏是"看起来专业"的最强信号;固定 ASL = 打卡式剪辑,是 AI 出片最大破绽(P0)。
- **音画 100ms**:超过约 100ms 唇同步/字幕延迟即被普通观众察觉。
- **响度 -14LUFS**:主流流媒体投放响度标准,避免平台二次压制导致忽大忽小。
- **LUT 强度 0.75**:拉满(100%)会让 LUT 盖过素材本身、丧失逐镜适配灵活性;70–80% 区间保留可调空间,故取 0.75(配 `{grading.lut-microtune}` 二次微调)。
- **踩拍比 0.30**:全片逐拍卡点会沦为"音乐可视化"、失去叙述;经验上约 30% 战略性强踩拍 + 70% 服从叙述最稳。

> **关于"哪些数值该 token 化"**:只有 **Lumeri 可调的身份/纪律旋钮**(饱和上限、色温容差、ASL、转场预算、LUT 强度、踩拍比等)进 token 层,由 `DESIGN.md` 单一定义。**通用电影手艺区间**(J-cut 1–3s、蒙太奇 10–30s、阅读速度 5–7 字/秒、建立镜头 2–4s 等)是行业常识而非本项目旋钮,**有意保留为正文**——硬 token 化只会让手册难读且伪造"可调"的假象。

## 五、机器校验(Lint)

沿用 Google 的 `@google/design.md` 思路,把可自动检查的部分工具化:

```bash
# 校验 token 引用完整性 + WCAG 对比度 + 结构
npx @google/design.md lint DESIGN.md

# 版本对比:检测 token / 理据回归(改版时用)
npx @google/design.md diff DESIGN.md DESIGN-v2.md
```

Lumeri 侧建议再补两个自定义检查(可挂 CI / commit 前钩子):
1. **硬编码数值扫描**:grep 所有 `.md`,凡出现裸数值(如 `36px`、`0.5s`)而非 `{token}` 引用 → 失败。
2. **红旗静态可查项**:字幕位置、转场计数、色彩空间声明等能从工程文件读出的项,自动跑第二节 A–F。

> 无法机器判定的项(情绪一致性、构图美感)由 Gemini 自评 + 建议交 **Antigravity 复审**(见文末)。
